# Edge Case Test Matrix (Current Status)

Date: 2026-04-09

This document maps each requested edge case to the current implementation status and what remains to be built.

Status legend:
- Working: currently supported by code and expected to pass
- Partial: some support exists, but behavior is limited
- Remaining: not implemented yet

---

## 1. Structural HTML Edge Cases

| Edge Case | Status | Current Behavior | What Remains |
|---|---|---|---|
| Shadow DOM | Remaining | HTML parsing is done from page content and BeautifulSoup. Closed/open shadow tree internals are not explicitly traversed. | Add Playwright DOM traversal for shadow roots and extract auth elements before BeautifulSoup fallback. |
| No form tag | Working | Detector falls back to nearest container (div/section/main) around password-like input if no form exists. | Add tests for more JS-only div-based auth UIs to confirm reliability. |
| Multiple forms | Partial | Logic finds password-related inputs and uses first matching parent form/container. Newsletter/search forms without password are naturally ignored. | Rank and return best candidate(s), optionally return top N forms with confidence. |
| Iframe logins | Remaining | Current scraper does not recurse into iframe document content for detection. | Enumerate frames in Playwright, evaluate each frame for auth signals, merge best result. |

## 2. Modern Frontend and Framework Edge Cases

| Edge Case | Status | Current Behavior | What Remains |
|---|---|---|---|
| Lazy loading login modal (requires click) | Partial | Playwright waits for input/form after page load, but no proactive click flow (for Sign In button) is executed. | Add interaction strategy: click likely auth triggers before final capture/detection. |
| Dynamic IDs and classes | Working | Detection relies on functional attributes (type, name, aria-label, placeholder, id, autocomplete) and not fixed class names. | Add more tests with autogenerated CSS class patterns. |
| SPA (client-side rendering) | Working | Requests fallback to Playwright when static HTML is minimal or lacks form/input; Playwright renders client-side UI. | Add SPA-specific regression tests for route transitions and delayed hydration. |

## 3. Authentication Variant Edge Cases

| Edge Case | Status | Current Behavior | What Remains |
|---|---|---|---|
| Single-field login step (email first, password later) | Remaining | Detection currently anchors on password-related fields. Email-only first step returns auth_found false. | Add multi-step auth heuristics and staged detection for identifier-first flows. |
| Social login only (Google/GitHub buttons) | Remaining | Without password/text-input auth signals, page is treated as no auth form. | Add button and OAuth provider pattern detection as alternate auth path. |
| Hidden honeypots | Partial | Input type hidden is skipped in extracted field list, but CSS-hidden decoys are not explicitly filtered. | Add visibility checks (computed styles, dimensions, aria-hidden) before counting fields. |

## 4. Negative and Error Edge Cases

| Edge Case | Status | Current Behavior | What Remains |
|---|---|---|---|
| Auth-protected redirect loops | Working | Requests maps TooManyRedirects to SCRAPE_REDIRECT_LOOP structured error. | Add direct integration test case to lock behavior. |
| No-auth informational page | Working | Returns auth_found false, html_snippet null, fields_detected empty, auth_confidence 0. | Add more no-auth pages to regression suite. |
| Bot protection (403 Forbidden) | Working | HTTP 403 maps to SCRAPE_BLOCKED structured error with actionable suggestion. | Add specific Cloudflare/Akamai scenario examples in tests/docs. |
| Malformed URL or empty input | Working | FastAPI request model rejects malformed URL with 422; empty URL blocked in frontend input validation. | Add explicit frontend test coverage if UI automation is introduced. |

## 5. Technical Delivery Edge Cases

| Edge Case | Status | Current Behavior | What Remains |
|---|---|---|---|
| Heavy markup / very large HTML | Partial | Requests and Playwright have timeout guards; detector parses full content; LLM prompt trims snippet size. | Add size-aware safeguards, parser budget limits, and profiling for large pages. |
| Encoding issues (non-UTF-8) | Partial | Requests provides decoded text via response handling, but no explicit charset correction pipeline is defined. | Add explicit encoding fallback logic and tests with non-UTF-8 fixtures. |

## Quick Summary

- Working now: no form tag fallback, dynamic class/id resilience, SPA rendering fallback, redirect loop handling, no-auth handling, 403 handling, malformed URL handling.
- Partial now: multiple forms ranking, lazy modal interaction, hidden honeypot visibility checks, heavy markup scaling, encoding robustness.
- Remaining: shadow DOM traversal, iframe login traversal, single-field staged logins, social-login-only detection.
